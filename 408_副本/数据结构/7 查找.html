基本概念

适合静态查找表的查找方法：顺序查找、折半查找、散列查找
适合动态查找表的查找方法：二叉排序树的查找、散列查找
平均查找长度：$\displaystyle\sum_{i=1}^{n} P_i*C_i$，其中$P_i$表示查找第i个元素的概率，$C_i$表示找到第i个元素所需进行的比较次数

顺序查找和折半查找

顺序查找

一般线性表的顺序查找

查找成功时，顺序查找的平均长度为$\displaystyle\sum_{i=1}^{n} P_i*(n-i+1)$
查找不成功时，ASl=n+1
对链表只能进行顺序查找

有序线性表的顺序查找

![[截屏2024-09-23 15.27.37.png]]
查找成功的ASl与一般线性表的顺序查找一样，查找失败的ASl=$n/2 + n/n+1$

折半查找

仅使用于有序的顺序表

折半查找的过程可用二叉树描述，称为判定树
![[截屏2024-09-23 15.39.03.png]]
折半查找的比较次数不会超过树的高度，树的高度=$\lceil log_2(n+1) \rceil$

![[截屏2024-09-23 15.39.43.png]]

分块查找（索引顺序查找）

将查找表分为若干子块，块内元素可以无序，但块间元素有序
![[截屏2024-09-23 15.43.36.png]]
平均查找长度最小为$\sqrt{n} + 1$

树型查找

二叉排序树

二叉排序树的删除
  1. 删除结点为叶结点，直接删除
  2. 删除结点为只包含左子树或右子树的结点，用子树直接替代
  3. 删除结点两个子树都包含，从右子树中找中序遍历的第一个结点，替代该结点
    ![[截屏2024-09-23 17.56.00.png]]
查找效率

若二叉排序树左右子树高度差小于1，平均查找长度$log_2{n}$
若二叉排序树只有单子树，平均查找长度为n
静态表用二分查找
动态表用二叉排序树

任意结点左右子树高度差不超过1

红黑树

  1. 每个结点只能是红色或黑色
  2. 根结点是黑色
  3. 叶结点都是黑色
  4. 不存在两个相邻的红结点（红结点的父亲和孩子都是黑色）
  5. 对每个结点，从该结点到任意一个叶结点的简单路径上，所含黑结点的数量相同
红黑树的插入
  1. 新插入的结点是红色的
  2. 新插入结点破坏了不红红和根叶黑特性才需要调整
  3. 新插入结点的叔叔结点是红色，新插入结点的叔、父、爷变色，把爷结点当作新插入的结点，重新判断
  4. 新插入的结点的叔叔结点是黑色，判断是什么类型（LL RR LR RL），根据平衡二叉树插入的规则，旋转，将中心点和旋转点变色
    注意：红黑树不是平衡二叉树，是二叉排序树，
    n个结点的红黑树最高$2*log_2{(n+1)}$

B树和B+树

B树

m阶B树时所有结点的平衡因子均等于0的m路平衡查找树

B树的特点和定义
1. 平衡：所有叶结点在同一行
2. 有序：结点内有序，任意元素的左子树小于他，右子树大于他
3. 多路：对于m阶B树，最多m个分支，m-1个元素
        根结点最少一个元素，两个分支

其他结点最少 $\lceil{m/2}\rceil$ 个分支，元素比分支少一个

B树的高度
对于n个关键字，高度为h，阶数为m的B树
h >= $log_2{n+1}$
h <= $log_{\lceil{m/2}\rceil}{((n+1)/2)} + 1$

B树的查找

与二叉排序树类似
注意：访问结点的操作在硬盘上进行，结点内的查找在内存中进行

B树的插入

先找到插入的位置进行插入，若没有上溢出，插入结束，若出现上溢出，则将$\lceil{m/2}\rceil$上移，两边元素分裂

B树的删除

删除的结点是非叶结点，用它的直接后继或直接前驱代替它，转换为删除叶结点
删除的结点是叶结点
未出现下溢，删除成功
出现下溢
兄弟结点够借，直接借
兄弟结点不够借，父结点下移，和一个兄弟结点合并

B+树

B+树的分支数等于元素个数
叶结点包含所有关键字，非叶结点仅起到索引作用

散列表

散列函数

直接定址法：H(key) = key 或 H(key) = a * key + b
除留取余法：H(key) = key % p
数字分析法
平方取中法：取关键字的平方值的中间几位作为散列地址

处理冲突的方法

开放定址法

线性探测法：冲突发生时，顺序查看表中下一个单元，直到找出一个空闲单元
线性探测法会出现堆积现象
平方探测法，也称二次探测法
双散列法：使用两个散列函数
伪随机序列法

拉链法

将所有的同义词存储在一个线性链表中

求查找失败的平均查找长度时，所有地址都要计算，无关键字的地址也要计算
